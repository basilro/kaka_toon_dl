📖 카카오웹툰 다운로드 — 사용 매뉴얼

설정 → 쿠키/경로 → 작품 등록 → 스케줄 실행. 아래 순서대로 따라 하면 됩니다.

1. 빠른 시작 (5단계)

  1. 쿠키 주입 — 설정 > 일반의 카카오 쿠키에 로그인 쿠키(JSON)를 붙여넣습니다. (→ 2번)
  2. 다운로드 경로 지정 — 다운로드 경로에서 저장할 폴더를 선택합니다.
  3. 작품 등록 — 체크할 웹툰에 작품을 한 줄에 하나씩 입력 (제목 / URL / contentId 모두 가능).
  4. 설정 저장 → 쿠키 검증 — 상단 설정 저장 후 쿠키 검증으로 로그인 상태를 확인합니다.
  5. 실행 — 즉시 실행으로 바로 받거나, 설정 > 자동 탭에서 스케줄러를 등록합니다. 진행 상태는 진행 상황 탭에서 확인합니다.
처음엔 작품 1개 + 즉시 실행으로 한 번 테스트한 뒤, 잘 받아지면 스케줄을 켜는 것을 권장합니다.

2. 카카오 쿠키 준비하기

이 플러그인은 사용자의 카카오 로그인 세션(쿠키)으로 카카오웹툰에 접근합니다. 비밀번호는 저장하지 않습니다.

  1. 크롬에서 https://webtoon.kakao.com 에 로그인합니다.
  2. 크롬 확장 프로그램 Cookie-Editor(또는 동등 확장)를 설치합니다.
  3. 카카오웹툰 탭에서 Cookie-Editor를 열고 Export → JSON 으로 복사합니다.
  4. 복사한 JSON을 설정 > 일반 > 카카오 쿠키 칸에 그대로 붙여넣고 설정 저장.
  5. 쿠키 검증 버튼으로 “쿠키 유효(로그인 상태 확인됨)”가 뜨면 완료.
필수 쿠키는 _kau(카카오 통합 로그인)입니다. 쿠키는 시간이 지나면 만료되며, 만료되면 다시 Export해서 주입해야 합니다.
웹훅을 설정해두면 쿠키 만료 시 1회 알림을 받을 수 있습니다. (→ 9번)

예) 홍도 · https://webtoon.kakao.com/content/홍도/1121 · 1121

기다무 1회 최대 장수기다무(자동 충전) 무료 티켓을 1회 실행에서 최대 몇 장 쓸지. 보통 1.

보유 일반 티켓 사용On이면 보유한 일반 이용권으로 유료 회차도 받습니다(잔량 전부). Off면 유료 회차는 스킵.
회차 폴더 ZIP 압축On이면 정상 완료된 회차 폴더를 NNNN_회차.zip으로 묶고 원본 폴더를 삭제합니다. 부분 실패 회차는 압축하지 않습니다. (→ 6번)
공지 기반 자동 다운유료화/종료 예정 공지에서 작품을 추출해 자동 다운로드. (→ 4번)
공지 다운 저장 폴더명공지 다운분이 들어갈 하위 폴더 이름. 기본 완결.
프록시 사용 / URL외부 프록시 경유. (→ 9번)
알림 탭

Discord 웹훅을 기본 대상으로 하며 Slack 웹훅도 자동 인식합니다. URL을 비워두면 해당 알림은 보내지 않습니다.

  • 쿠키 만료 알림 URL — 쿠키 만료를 감지하면 1회 발송(복구되면 다음 만료 때 다시 1회).
  • 다운로드 완료 알림 URL — 자동 실행이 끝나고 받은 회차가 있을 때 요약 발송. 받은 게 없으면 안 보냄.
각 URL 옆 테스트 버튼으로 수신 여부를 먼저 확인하세요.
자동 탭
  • 스케쥴링 실행 정보 — Interval(분) 또는 Cron. 예) 180 또는 0 */3 * * * (3시간마다).
  • 시작시 자동실행 — On이면 SJVA 시작 시 스케줄러를 자동 등록.

4. 자동 다운로드 동작 방식

스케줄러(또는 즉시 실행)가 돌면 체크할 웹툰의 각 작품에 대해 받지 않은 회차를 찾아 다운로드합니다. 회차 종류별 처리는 다음과 같습니다.

구분처리
무료제한 없이 받습니다.
보유이미 열람권이 있는 회차 — 받습니다.
기다무기다무 무료 티켓 사용 On일 때, 충전된 티켓 한도(1회 최대 장수) 내에서 받습니다.
유료보유 일반 티켓 사용 On이고 보유 티켓이 있을 때만 받습니다. 없으면 스킵.
잠금받을 수 없음 — 스킵.
공지 기반 자동 다운 (유료화/종료 예정)

공지 기반 자동 다운을 켜면, 자동 실행 시 카카오웹툰 공지의 “YYYY년 M월 유료화 & 종료 작품” 안내에서 작품을 추출해 받습니다. 별도 스케줄 없이 기존 스케줄에 통합됩니다.

  • 받는 범위: 무료 + 보유 회차, 그리고 보유 티켓이 있는 유료 회차.
  • 기다무 티켓은 소진하지 않습니다(예정작에 티켓을 낭비하지 않도록).
  • 저장 위치는 다운로드경로/완결/... (폴더명은 공지 다운 저장 폴더명).
  • 작품을 검색으로 못 찾는 등 실패 시, 웹훅이 설정돼 있으면 실패 작품을 알림으로 보냅니다(중복 방지).

5. 수동 다운로드

수동 다운로드 탭은 작품 하나를 즉석에서 골라 받을 때 씁니다.

  1. 작품/회차 URL(또는 contentId)을 입력하고 분석.
  2. 회차 목록이 뜨면 받을 회차를 체크합니다. 무료·미수신 회차만 선택 가능(유료/잠금/이미 받음 제외).
  3. 선택 다운로드를 누르면 직렬로 받기 시작합니다. 취소로 중단할 수 있습니다.
뷰어 URL .../viewer/{contentId}/{episodeId} 형식을 넣으면 그 회차만 자동 선택됩니다.
수동 다운로드는 사용자가 직접 누르는 동작이라 자동/공지 다운로드와는 독립적으로 동작합니다. 단, 압축·자동 다운 등 다른 작업이 돌고 있으면 충돌 방지를 위해 끝난 뒤 다시 시도하라는 안내가 뜹니다.

6. 저장 구조와 중복 처리

폴더 구조
다운로드경로/
└─ 작품제목/
   ├─ info.xml          (작품 메타)
   ├─ cover.jpg         (표지, 없으면 생략)
   ├─ 0001_회차제목/     (압축 Off)
   │   ├─ 001.webp
   │   └─ 002.webp ...
   └─ 0002_회차제목.zip  (압축 On — 폴더 대신 zip)

다운로드경로/완결/작품제목/...   ← 공지 기반 다운분

회차 폴더 이름은 NNNN_회차제목(4자리 0채움)입니다.

이미 받은 회차는 어떻게 판단하나?
  • 기본 판단 기준은 DB의 다운로드 이력(회차 단위 completed)입니다. 이력에 완료로 있으면 다시 받지 않습니다.
  • 압축(ZIP) 사용 시: DB 이력에 없더라도 다운로드 폴더에 해당 회차 zip이 이미 있으면 다시 받지 않고 이력에만 인식(등록)합니다. → 같은 회차를 헛다운로드하거나 유료 티켓을 다시 쓰는 일을 막습니다.
  • zip은 정상 완료된 회차에만 만들어지므로, “zip 있음 = 온전히 받은 회차”로 안전하게 간주합니다. 받다 만 폴더(zip 없음)는 다시 받습니다.
  • 압축 처리 중 이미 같은 이름의 zip이 있으면 덮어쓰지 않고 기존 zip을 유지합니다.
처음 도입해서 “예전에 받아둔 zip”을 이력에 인식시키고 싶을 때, 압축을 켜둔 상태로 자동/공지 실행을 한 번 돌리면 해당 작품의 기존 zip들이 자동으로 이력에 등록됩니다(재다운로드 없이).
압축을 끄고 폴더로 보관하는 경우엔 이 “디스크 인식”이 동작하지 않습니다(폴더만으론 완전 다운 여부를 보장할 수 없어서). 중복을 디스크로 방지하려면 압축을 켜는 것을 권장합니다.

버튼동작
설정 저장변경한 설정을 저장.
즉시 실행스케줄을 기다리지 않고 자동 다운로드를 지금 1회 실행.

8. 진행 상황 · 다운로드 이력

  • 진행 상황 — 자동/수동 다운로드의 현재 상태, 작품·회차·페이지 진행률을 실시간(1.5초 폴링)으로 보여줍니다.
  • 다운로드 이력 — 받은 회차 목록. 상태(완료/부분/실패/스킵 등) 필터와 제목 검색을 지원하고, 선택 항목을 삭제할 수 있습니다.

9. 알림(웹훅) · 프록시

웹훅 알림
  • Discord 웹훅 URL 예: https://discord.com/api/webhooks/{ID}/{TOKEN} (Slack 웹훅도 자동 인식).
  • 쿠키 만료 알림 / 다운로드 완료 요약 알림을 각각 켤 수 있습니다. (설정 > 알림)
프록시 (선택)
  • 프록시 사용 On + URL 값이 있을 때만 적용됩니다. 예) http://user:pass@host:8080 · socks5h://host:1080.
  • On이면 API 호출·이미지 CDN·표지 다운로드가 모두 같은 프록시를 사용합니다(토큰 IP 검증 회피).
  • 트래픽 부담이 커지므로, 불안정하면 끄세요.

10. 문제 해결 (FAQ)

받아지는 게 0개예요 / “쿠키 만료” 가 떠요

쿠키가 만료됐을 가능성이 큽니다. 카카오웹툰에 다시 로그인 → Cookie-Editor로 재-export → 쿠키 칸에 붙여넣고 설정 저장 → 쿠키 검증으로 확인하세요.

유료 회차가 안 받아져요

보유 일반 티켓 사용이 켜져 있는지, 그리고 해당 작품에 실제 보유 티켓이 있는지 확인하세요. 티켓이 없으면 유료 회차는 스킵됩니다(이 플러그인은 티켓을 구매하지 않습니다).

잠금/유료 회차를 받게 하려면?

기다무 회차는 기다무 무료 티켓 사용을, 보유 티켓이 있는 유료 회차는 보유 일반 티켓 사용을 켜세요. 카카오 정책상 권한이 없는 회차는 받을 수 없습니다.

같은 회차를 자꾸 다시 받아요

압축(ZIP)을 켜두면 디스크의 기존 zip을 인식해 재다운로드를 피합니다. (→ 6번) 폴더 보관(압축 Off) 상태에서는 디스크 인식이 동작하지 않습니다.

스케줄이 안 도는 것 같아요

설정 > 자동에서 스케줄러가 등록·실행 중인지 확인하고, 로그 탭에서 실행 로그를 확인하세요. 즉시 실행으로 수동 트리거해 동작 여부를 먼저 점검할 수 있습니다.

.webp.cef / _keys.json 파일이 생겼어요

이미지 복호화에 실패한 페이지가 있을 때만 원본(.webp.cef)과 키 파일(_keys.json)을 남깁니다(정상 회차엔 안 생김). 대부분 일시적 오류이니 해당 회차를 이력에서 지우고 다시 받아 보세요. 과거 잔재는 _keys.json 정리로 청소할 수 있습니다.

※ 이 플러그인은 본인이 정당하게 접근 권한을 가진 콘텐츠를 개인 보관 목적으로 내려받는 용도입니다. 카카오 이용약관과 저작권법을 준수해 사용하세요.
